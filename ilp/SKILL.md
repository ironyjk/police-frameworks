# Intelligence-Led Policing — 데이터로 경찰을 경영하기

> Background and theory: Read references/foundation.md

---

## 적용 단계

### Phase 1 — 조직 준비
- 분석 역량 확보 (인력 or 외부 협력)
- 데이터 접근 권한 정리
- 의사결정 회의체 설립 (TCG 모델)

### Phase 2 — 첫 Strategic Assessment
- 전년도 데이터 종합
- 상위 위협 5~10개 식별
- 자원 배분 원칙 수립

### Phase 3 — Tactical Cycle
- 월별 분석 → 태스킹 회의 → 배치 → 피드백
- 최소 6개월 운영해야 문화 정착

### Phase 4 — Impact Review
- 분기별 어떤 결정이 효과 있었는지 검토
- 없으면 접근 변경

## 지역 소통 창구 활용 (선택 사항)

경찰이 ILP를 운영할 때, **지역 소통 창구**(경찰발전협의회·주민자치위·지역치안협의체 등)에 분석 결과와 우선순위 근거를 공유하면 다음 효과가 있습니다 — 단, 분석·결정 주체는 어디까지나 경찰입니다:

1. **데이터 해석의 투명화** — 경찰이 어떤 데이터에 기반해 결정하는지 지역사회가 인지
2. **우선순위의 공공성** — 상습자/반복피해자 선정 기준을 공개 가능한 언어로 설명
3. **Impact 검증** — 경찰 자체 평가 외에 지역 체감 피드백 수렴
4. **차별 방지** — 특정 계층/지역에 대한 부당한 집중이 외부에도 보이는지 점검

중요한 것은: 이 소통은 ILP의 **부가 장치**이지, 소통 조직이 ILP를 수행하는 것은 아닙니다.

---

## 자주 하는 실수

1. **분석 보고서만 생산, 결정 없음** — Interpret만 있고 Influence 없음
2. **기술 중심 오해** — ILP는 GIS 프로그램이 아니라 **관리 철학**
3. **"빅데이터"로 착각** — 엄청난 데이터 필요 없음. 기본 신고 데이터로도 시작 가능
4. **상습자 프로파일링 = 차별** 함정 — 절차적 정의와 법적 정당성 확보 필수
5. **지휘부 변경 시 붕괴** — 제도화되지 않으면 특정 개인 의존

## 언제 쓸 것인가

**적합:**
- 자원 부족 하에 우선순위 결정 필요
- 상위 전략 수립
- 관할 단위 경영
- 경찰서·지구대의 데이터 기반 전환

**부적합:**
- 개별 사건 수사 현장
- 단일 민원 대응
- 긴급 상황
