# /peel 샘플 테스트 — 경주 지역 경찰 실무 시나리오

> 이 문서는 `/peel` 메타 라우터가 **일선 경찰 실무자**(지구대·파출소·경찰서 생활안전계)의 반복 민원 분석 안건을 어떻게 처리하는지 보여주는 시뮬레이션입니다. 입력은 가상의(그러나 경주 지역 맥락에서 있을 법한) 시나리오이며, 출력은 라우터가 12개 프레임워크 중 어떤 것을 선택·시퀀싱하고 각 단계에서 무엇을 권장하는지 단계별로 제시합니다.
>
> **참고**: 이 도구는 소프트웨어 개발자인 최희철(경주경찰서 경찰발전협의회 회원)이 참고용 자료로 제작한 것이며, 실제 분석·판단·개입은 경찰의 몫입니다. 경찰발전협의회는 봉사단체로서 범죄 분석·수사에는 관여하지 않습니다.

---

## 시나리오 (입력)

**상황 출처**: 성건파출소(경주경찰서 소속)에 접수된 반복 민원 패턴

```yaml
situation: |
  경주시 성건동 ○○근린공원(주거지 내부)에서 지난 3개월간 야간 시간대
  (22~02시)에 청소년 집단(15~18세 추정, 5~10명) 음주·흡연·소란 민원이
  반복 접수되고 있다. 주민 민원 총 14건.

  순찰차가 접근하면 흩어지고, 떠나면 다시 복귀하는 패턴.
  주변 편의점에서 청소년 주류 판매 의심 정황도 있음.

  일부 주민은 "무관용 단속"을 강하게 요구하고 있고,
  다른 주민은 "우리 애들인데 범죄자 취급하지 말라"는 입장.
  성건파출소 인력으로는 야간 상시 배치 불가능.

problem_type: 반복 패턴 + 장소 집중 + 사람 요소
location: 경주시 성건동 ○○근린공원 (주거지 내부)
people: 청소년(가해 추정), 주민(피해 호소), 편의점주(제3자)
requester: 성건파출소장 · 경주경찰서 생활안전계
urgency: 주의 (지속되면 위험)
constraints: 야간 인력 부족, 주민 의견 분열, 단속 시 인권 이슈
goal: 장기적 문제 해결 + 주민 신뢰 유지 + 청소년 낙인 방지
```

> **왜 성건동인가** (시나리오 설계 근거): 성건동은 경주고·경주여고·경주중앙중 등 학교가 인접해 있고, 주거·상권·학교가 섞인 전형적 생활권이다. 성건파출소 관할로 이런 청소년 집중 민원이 실제로 접수될 만한 지역이다. 특정 공원을 지칭하지 않기 위해 "○○근린공원"으로 표기한다.

## 라우터 출력

### 1단계: 상황 분류

**문제 유형:**
- 반복 패턴 (3개월, 14건) → 축 1 (문제지향 경찰활동)
- 장소 집중 (특정 공원) → Hot Spots 요소
- 환경 요인 가능성 (공원 설계·조명·시설) → CPTED 요소
- 사람 요소 (청소년 + 편의점 + 주민) → 축 4 (지역사회·정당성)
- "무관용 단속" 요구 → **Broken Windows 비판적 검토 필수**
- 위기·조사 단계 아님 → 축 2/3 전체 제외

**제외된 프레임워크:**
- PEACE, Cognitive Interview — 조사 면담 단계 아님
- BCSM, ICAT — 현장 위기 아님
- ILP — 단일 지점 전술 문제이므로 전면 전략 수준 아님

**선정된 프레임워크:**
- **SARA** (전체 뼈대)
- **Crime Triangle** (원인 분해)
- **Hot Spots** (공간 집중 확인)
- **CPTED** (환경 개입)
- **COP** (주민·기관 협력)
- **Procedural Justice** (청소년·주민 대응 원칙)
- **Broken Windows** (비판적 필터 — "무관용" 요구 검증)

### 2단계: 실행 파이프라인

```
Phase 1 — 문제 정의 (Scanning)
  └ SARA-S + Hot Spots
  [주체] 성건파출소 담당, 경주경찰서 범죄예방진단팀 또는 분석 가능 인력

Phase 2 — 원인 분석 (Analysis)
  ├ Crime Triangle (6박스 분해)
  └ CPTED 현장 점검
  [주체] 지구대장 + 경주경찰서 생활안전계 + (필요 시) 경주시 공원녹지과

Phase 3 — 개입 설계 (Response)
  ├ Broken Windows 비판적 필터 (먼저 "무관용 아님" 확정)
  ├ CPTED 환경 개입
  ├ COP 주민·기관 파트너십
  └ Procedural Justice 원칙 삽입
  [주체] 경찰서 생활안전과장 승인 하 지구대 실행

Phase 4 — 평가 (Assessment)
  └ SARA-A 사이클
  [주체] 생활안전계 자체 평가 + 3개월 후 재측정
```

## 각 프레임워크 적용

### Phase 1 — SARA-Scanning + Hot Spots

**수행 주체:** 성건파출소 담당 경찰관

**목표:** "치안 문제"가 아닌 **구체적 문제**로 정의

**결과물:** 한 줄 문제 정의
> "경주시 성건동 ○○근린공원의 북측 놀이터·정자 구역에서, 목~토요일 22~02시 사이, 15~18세 청소년 5~10명이 음주·흡연·소란 행위를 반복하여 인근 주거동 주민들에게 수면·생활 피해를 주고 있다."

**Hot Spots 확인 방법:**
- KICS에서 해당 공원 반경 지난 3개월 112 신고 추출
- 좌표·시간대 분포 → 공원 전체가 아닌 **북측 놀이터·정자 집중** 여부 확인
- 요일·시간대 패턴: 목~토 22~02시 집중 여부 확인
- 순찰 배치의 질문은 "공원 전체 순찰"이 아닌 "해당 구역 × 해당 시간 집중"

**산출물:** 문제 정의서(1페이지), 핫스팟 지도, 시간대 분포표 — 생활안전계장 보고

---

### Phase 2 — Crime Triangle (6박스 분해)

**수행 주체:** 지구대장 + 생활안전계 담당

| 박스 | 현재 상태 | 개입 가능성 |
|---|---|---|
| **Offender (청소년)** | 5~10명, 인근 학교 학생 추정 | 단속형 접근은 낙인·소송 위험 |
| **Handler (가해자 통제자)** | 부모·학교 미인지 상태 가능 | **강력 개입 지점** — 학교·가정 연계 |
| **Target (주민 수면)** | 인근 주거동 | 직접 개입 어려움 (피해 이동 불가) |
| **Guardian (대상 보호자)** | 경찰 순찰 일시 효과만 | 한계 명확 |
| **Place (공원 북측)** | 조명 미흡 추정, 정자 음영 공간 | **CPTED 개입 지점** |
| **Manager (공원 관리자)** | 경주시 공원녹지과 | **행정 협력 지점** |

**가장 움직이기 쉬운 박스:** Handler, Place, Manager
**가장 움직이기 어려운 박스:** Offender (단속 중심), Target, Guardian

**→ 개입은 Handler + Place + Manager에 집중**

---

### Phase 3a — Broken Windows 비판적 필터

"무관용 단속"을 요구하는 주민 목소리에 대해 지구대장·생활안전계가 사전 검토:

**체크리스트:**
- [x] 문제가 **환경**인가 **사람**인가? → **둘 다**, 그러나 환경 개입이 더 효과적·지속가능
- [x] 제안된 조치가 **특정 집단에 차별적 영향**을 주는가? → **예** (청소년 대상 집중 단속)
- [x] 절차적 정의 4요소 부합? → 현재 요구는 Voice·Neutrality·Respect 훼손 우려
- [x] 근본 원인(청소년 여가·공간 부재, 편의점 판매) 접근? → **단속만으로는 미흡**
- [x] "눈에 안 보이게" 쫓아내는 것 vs 문제 해결? → **이동 가능성 높음** (인근 다른 공원으로)

**결론:** 단순 집중 단속(Zero Tolerance)은 이 사안의 정답이 아니다. 대신 **CPTED + 학교·가정 연계 + 행정 협조**의 복합 접근이 필요하다. 단속 요구 주민에게도 "왜 단속만으로는 해결되지 않는가"를 설명할 필요가 있다.

**경찰이 주민에게 전달할 수 있는 핵심 메시지:**
> "단속하면 이 아이들은 이 공원을 떠나 다른 공원으로 갑니다. 3개월 후 우리는 지금과 똑같은 민원을 다른 동에서 받게 됩니다. 그리고 이 아이들과 그 가족은 평생 경찰을 신뢰하지 않는 시민이 됩니다. 저희는 단속을 안 하려는 게 아니라, 단속만으로는 문제가 해결되지 않아서 다른 방법을 병행하는 겁니다."

---

### Phase 3b — CPTED 현장 점검

**수행 주체:** 경주경찰서 범죄예방진단팀(또는 생활안전계) 주도, 경주시 공원녹지과와 공동

**북측 놀이터·정자 구역 체크리스트:**

| 6대 원칙 | 현재 상태 (추정) | 개입안 |
|---|---|---|
| 자연감시 | 조명 부족, 조경 키 큼 | 조명 교체, 조경 가지치기 |
| 접근통제 | 야간 공원 개방 | 22시 이후 북측 일부 동선 조정 검토 |
| 영역성 | 관리주체 표시 없음 | 관리주체 안내판, 커뮤니티 게시판 |
| 활동지원 | 야간 합법 활동 부재 | 저녁 운동·동호회 유치 (자연감시 확대) |
| 유지관리 | 청소 주기 확인 필요 | 민원 핫스팟 구역 청소 빈도 상향 |
| 표적경화 | 해당 없음 | — |

**현장 점검팀 구성:**
- 성건파출소 담당 경찰관
- 경주경찰서 범죄예방진단팀 (또는 생활안전계)
- 경주시 공원녹지과 담당자
- 주민 대표 1~2명 (현장 증언)

---

### Phase 3c — 파트너십 설계 (COP)

**조정 주체:** 성건파출소장 / 경주경찰서 생활안전과

**파트너 매핑:**

| 주체 | 역할 |
|---|---|
| 성건파출소 | 현장 대응, 핫스팟 순찰 (Koper 15분 규칙) |
| 경주경찰서 여성청소년과 | 청소년 접촉·상담·선도 |
| 경주시 공원녹지과 | 조명·조경·시설 예산·시공 |
| 인근 중·고교 | Handler 역할 (학생 인지·상담) |
| 경주시 청소년상담복지센터 | Handler + 대안 활동 제공 |
| 인근 편의점주 협의 | 청소년 주류 판매 차단 |
| 주민자치위원회 | 주민 소통 창구 |
| 학부모회 | 가정 연계 |

**초기 조정 회의(주재: 지구대장 또는 생활안전계장)의 목표:**
1. 문제 정의 합의 (데이터·피해·원인)
2. Broken Windows 함정 공유 (무관용 아닌 이유)
3. 각자 역할 분담 및 연락처 명확화
4. 3개월 실행 + 3개월 평가 합의

---

### Phase 3d — Procedural Justice 원칙 삽입

**현장 청소년 대응 시 (성건파출소 일선 경찰관):**
- **Voice**: 발견 시 단순 해산이 아닌, 이름·상황 듣기 ("무슨 일로 모였니?")
- **Neutrality**: 동일 사안에 동일 대응, 감정적 처벌 배제
- **Respect**: 존칭 사용, 공개 망신 금지, 부모 연락 시에도 아이 동반해서 설명
- **Trustworthiness**: "너희를 미워해서가 아니라 주민 피해와 너희 미래 둘 다 걱정해서"

**주민 대응 시 (특히 단속 요구 주민):**
- **Voice**: 지구대·생활안전계가 분석 결과와 대안을 설명할 간담회 마련
- **Neutrality**: "단속을 안 해서가 아니라, 단속만으로는 안 되기 때문"
- **Respect**: 주민의 피해 감정 인정 먼저
- **Trustworthiness**: 실행 계획과 평가 시점을 구체적으로 약속

---

### Phase 4 — Assessment (3개월 후)

**수행 주체:** 경주경찰서 생활안전계

**평가 지표:**
- 1차 지표: ○○근린공원 관련 야간 민원 건수 (KICS 기준)
- 2차 지표: 인근 지역(범죄 이동 확인) 민원 건수
- 3차 지표: 주민 체감 (간이 설문, 주민자치위 협조)
- 4차 지표: 청소년·학부모 반응 (청소년상담복지센터 경유)

**결과별 다음 단계:**
- **감소 + 이동 없음** → 개입 유지, 다른 핫스팟에 확장
- **감소 + 이동 있음** → Crime Triangle 재분석, 이동 지점 선제 개입
- **감소 없음** → Analysis 단계 가설 오류 가능성, SARA 사이클 재시작

---

## 라우터가 전달하는 경고

1. **단순 순찰 강화로 해결하려 하지 말 것** — 야간 상시 배치 불가, 이탈 즉시 복귀
2. **"무관용 단속" 요구에 동조하지 말 것** — 낙인·이동·신뢰 훼손
3. **학교·가정 연계 없이 경찰 단독으로 하지 말 것** — Handler 박스가 핵심
4. **청소년 대응이 주민 대응보다 더 신중해야 함** — 평생의 경찰 불신자 생성 위험
5. **평가 없이 종료하지 말 것** — SARA의 Assessment 단계는 필수

---

## 이 도구의 위치

이 `/peel` 출력은 **경찰 실무자가 의사결정을 하기 위한 첫 번째 분석 초안**입니다. 경찰은 이 초안을 KICS 실데이터, 현장 지식, 조직 맥락, 법적 권한과 결합해 최종 판단을 내립니다.

작성자 최희철은 경주경찰서 경찰발전협의회 회원이자 소프트웨어 개발자로서, 해외 학술 프레임워크를 한국 경찰 실무 맥락에 맞게 정리한 참고 자료를 제공하는 역할입니다. 경찰발전협의회는 봉사·지원 단체이므로 실제 범죄 분석·수사·집행에는 관여하지 않으며, 이 도구 역시 경발협 자체의 활동 도구가 아니라 **경찰 실무에 드리는 참고 자료**입니다.

## 이 시뮬레이션의 한계

- 실제 공원·조명·CCTV 상태는 현장 점검 없이 알 수 없음 — CPTED 항목은 가설
- 청소년의 실제 구성·동기는 상담 없이 추정 불가 — Handler 연계 전제
- 신고 14건의 세부 내용이 KICS에서 분석되면 Crime Triangle이 더 정교해짐
- 주민 내부 세력 관계는 COP 설계의 핵심 변수 (이 시뮬레이션에서는 일반화)
- 성건동 ○○근린공원은 **가상 설정**이며, 실제 민원 분포는 경주경찰서 내부 데이터에서만 확인 가능

**라우터는 첫 번째 가설일 뿐이며, 실제 적용은 경찰이 현장 데이터와 주민 대화로 갱신해야 합니다.**
