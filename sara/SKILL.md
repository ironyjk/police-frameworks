# SARA — 문제지향 경찰활동의 심장

> "경찰의 본질적 업무는 신고 대응이 아니라 문제 해결이다." — Herman Goldstein, 1979

## 배경

1970년대 후반 Herman Goldstein은 당시 미국 경찰의 현실을 관찰하고 하나의 불편한 사실을 지적했습니다: **경찰은 같은 곳에, 같은 사람에게, 같은 신고를 받으러 반복해서 출동한다.** 대응은 효율화되었지만 문제는 줄지 않았습니다.

Goldstein의 제안은 단순했습니다. 경찰은 개별 사건(incidents)이 아니라 **문제(problems)** 를 다뤄야 한다. 문제란 시간·공간·집단에 걸쳐 반복되는, 서로 연결된 사건들의 집합이다.

1987년 Eck와 Spelman은 이 철학을 실행 가능한 프로세스로 만들었습니다 — **SARA**.

---

## 4단계

### 1. Scanning (탐색)

**목표**: 단순한 사건의 나열을 넘어 **문제**로 식별할 것

**해야 할 일:**
- 반복 신고, 민원, 범죄 통계에서 패턴을 찾습니다
- 다음 기준 중 하나 이상에 해당하면 "문제"로 분류합니다:
  - 동일 장소에 반복 출동 (예: 같은 주소 월 5회 이상)
  - 동일 피해자/가해자 반복
  - 동일 유형의 피해(물건, 금액, 방식)
  - 지역 주민·이해관계자의 지속적 우려
- 문제를 한 줄로 정의합니다: "누가, 언제, 어디서, 어떤 방식으로, 어떤 결과를 겪는가"

**피해야 할 것:**
- "치안 악화", "범죄 증가" 같은 모호한 표현
- 신고 건수만 보고 원인을 추정

**한국 경찰 적용 예:**
> "○○역 3번 출구 주변에서 금요일 22시~02시 사이, 20대 남성 주취자 간 시비·폭행 신고가 최근 6개월 월평균 8건 발생. 반복 당사자 3명 확인."

이것이 '문제'입니다. "역 앞 치안 문제"가 아닙니다.

---

### 2. Analysis (분석)

**목표**: 문제의 **원인**을 이해할 것 — 무엇이 이 문제를 반복 가능하게 만드는가?

**도구:** 범죄 분석 삼각형(crime-triangle 스킬 참조)
- **가해자(Offender)**: 누구인가? 왜 이곳을 선택하는가? 억제 수단이 있는가?
- **대상/피해자(Target/Victim)**: 무엇이/누가 공격받는가? 매력적 이유는?
- **장소(Place)**: 공간적 특징은? 관리자가 있는가? 자연감시가 되는가?

**4W1H 질문:**
- **Who** — 가해자와 피해자는 누구이며 어떤 관계인가?
- **What** — 정확히 어떤 행위가 반복되는가?
- **When** — 요일, 시간대, 계절성 패턴
- **Where** — 정확한 공간 분포, 주변 시설
- **How** — 어떤 수단·방법·기회를 이용하는가?

**데이터 소스:**
- 112 신고 로그 (시공간 분포)
- 현장 보고서 (정성적 맥락)
- 피해자 면담
- CPTED 현장 점검
- 주민·상인 증언
- 과거 대응 기록과 결과

**가설 수립 → 검증**

분석 단계의 핵심은 **"왜 이 문제가 여기서 계속 일어나는가"**에 대한 검증 가능한 가설을 세우는 것입니다.

예: "조명이 어두워서" vs "특정 술집이 폐점 후 시비 배출구 역할을 해서" — 어느 가설이 데이터와 더 일치하는가?

---

### 3. Response (대응)

**목표**: 분석 결과에 기반한 **맞춤형 개입** — 일반적 "순찰 강화"가 아님

**개입 옵션의 스펙트럼:**

1. **경찰 내부 조치**
   - 타깃 순찰(핫스팟에 집중)
   - 탐문 수사
   - 관련자 경고·접근금지

2. **환경 조치 (CPTED)**
   - 조명, 시야, CCTV 재배치
   - 출입통제, 영역 구분

3. **제3자 활용**
   - 건물 소유주, 상가 협회
   - 관할 구청, 시설관리공단
   - 학교, 복지시설

4. **정책·제도 개선**
   - 영업시간 조례
   - 보건소·복지 연계
   - 지역 조례 제정 건의

**SARA의 핵심 원칙: 경찰 자원은 개입의 한 옵션일 뿐이다.**

많은 문제는 경찰이 직접 해결할 수 없습니다 — 예산 없고, 권한 없고, 지속가능성 없음. 지속적 해결은 대개 제3자(건물주, 관리소, 지자체)의 협조를 통해 이뤄집니다.

**대응 설계 체크리스트:**
- [ ] 분석 결과와 직접 연결되는가?
- [ ] 경찰 외 주체의 협조가 필요한가? 확보했는가?
- [ ] 비용/노력 대비 효과가 "순찰 강화"보다 우월한가?
- [ ] 평가 가능한 지표가 설정되어 있는가?
- [ ] 의도하지 않은 부작용(범죄 이동, 차별 민원)을 예측했는가?

---

### 4. Assessment (평가)

**목표**: 대응이 **실제로** 문제를 줄였는지 측정 — 단순 "느낌"이 아니라 데이터로

**평가 지표:**
- **사전-사후 신고 건수** (최소 3개월 이전/이후)
- **문제의 이동 여부** (인접 지역으로 옮겨갔는지)
- **피해자·주민의 체감 변화**
- **의도한 메커니즘이 실제로 작동했는지**

**평가 결과의 3가지 시나리오:**
1. 문제 해결됨 → 개입 유지 또는 축소
2. 일부 감소 → 분석으로 돌아가 보완
3. 변화 없음 → **분석 단계의 가설이 틀렸을 가능성** — SARA 사이클 재시작

**평가는 종착역이 아니라 다음 Scanning의 시작점입니다.**

---

## SARA를 잘못 쓰는 대표적 실패

1. **S 생략**: "이런 문제가 있다고 들었어" — 데이터 없이 바로 분석
2. **A 대충**: 원인 분석 없이 "일단 순찰 배치"
3. **R 경찰 단독**: 제3자 협조 없이 경찰 자원으로만 해결 시도 → 지속불가
4. **Assessment 없음**: 대응 후 재측정 없이 "잘 됐다"고 판단
5. **SARA를 일회성으로 봄**: 한 번 돌리고 끝. 실제로는 반복 사이클

## 한국 경찰 적용 사례 (가상)

**문제:**
○○시 ○○동 주상복합단지에서 3개월간 심야시간 주차장 내 차량 접촉·물품 절도 신고 14건.

**Scanning:**
- 신고 시간대: 23시~04시 집중
- 신고 지점: 지하 2층 동측 구역 11건
- 유사 수법: 유리창 파손 없이 문손잡이 조작 흔적

**Analysis:**
- 가해자: 내부 구조를 아는 인물 가능성 (외부 침입 흔적 적음)
- 대상: 특정 차종 집중 없음, 차량 내 보관 현금/귀중품 중심
- 장소: 해당 구역 CCTV 사각, 조명 LED 수명 만료, 출입 차단기 미작동
- 가설: 물리적 접근 통제 실패 + 자연감시 부재

**Response:**
- CCTV 사각 해소 (관리사무소 협조)
- 조명 교체 (구청 도시개선 예산 활용)
- 입주민 대상 "차량 귀중품 미보관" 캠페인
- 순찰은 최소화, 해결은 관리소 주도

**Assessment:**
- 3개월 후 신고 1건 (93% 감소)
- 인접 동 주차장에서 신고 2건 → 범죄 이동 의심, SARA 사이클 재시작

## 언제 SARA를 쓸 것인가

**적합:**
- 3개월 이상 지속되는 반복 문제
- 민원이 누적되는 이슈
- "순찰 강화"로 해결되지 않은 문제
- 지구대·경찰서 생활안전계의 반복 민원 구조 분석

**부적합:**
- 단일 긴급 사건 (ICAT/BCSM 사용)
- 즉각 법 집행이 필요한 상황
- 분석 시간이 없는 현장 판단

---

## 함께 쓰는 프레임워크

- **Crime Triangle** — SARA의 A 단계에서 필수
- **CPTED** — 환경 개입이 R 단계 옵션일 때
- **Hot Spots** — 지리적 집중이 명백할 때
- **ILP** — 데이터 분석 역량이 조직에 있을 때
- **Procedural Justice** — 개입 실행 시 항상 삽입
