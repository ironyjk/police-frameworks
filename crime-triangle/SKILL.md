# 범죄 삼각형 — 왜 범죄가 발생하는가

> "범죄는 동기만으로 발생하지 않는다. 기회가 있어야 한다." — Cohen & Felson, 1979

## 이론적 배경

1979년 Lawrence Cohen과 Marcus Felson은 전통적 범죄학의 가정에 의문을 던졌습니다: 전통 이론은 범죄를 **가해자의 동기**로 설명했지만, 그들은 동기만으로는 부족하다고 봤습니다.

그들의 관찰은 단순했습니다. 전후 미국에서 생활수준이 올랐는데도 범죄율이 **오히려 증가**했습니다. 기존 이론으로는 설명이 안 됐습니다.

**일상활동이론(Routine Activity Theory):** 범죄가 발생하려면 세 가지가 시공간에서 만나야 합니다:

1. **동기화된 가해자** (Motivated Offender)
2. **적절한 대상** (Suitable Target)  
3. **유능한 감시자의 부재** (Absence of Capable Guardian)

전후 미국: 여성 노동시장 진출 → 낮 시간 빈집 증가 → 감시자 부재 → 주거 침입 증가. 동기는 그대로였지만 **기회 구조**가 바뀐 것입니다.

---

## 문제 분석 삼각형 (확장판)

John Eck과 Ronald Clarke는 일상활동이론을 경찰 현장에 맞게 확장해 **문제 분석 삼각형**을 만들었습니다. 3요소에 각각의 **억제자**를 추가한 구조입니다.

```
            [가해자]
            Offender
           /        \
          /          \
  [Handler]        [장소]
  (가해자 억제자)    Place
                     |
                [Manager]
                (장소 관리자)
          [대상/피해자]
          Target/Victim
               |
          [Guardian]
          (대상 보호자)
```

### 내부 삼각형 (3요소)

| 요소 | 설명 | 예 |
|---|---|---|
| **Offender (가해자)** | 범죄를 저지를 동기와 능력을 가진 자 | 전과자, 조직, 충동적 가해자 |
| **Target (대상)** | 공격 대상이 되는 사람·물건·장소 | 현금, 차량, 취약 피해자 |
| **Place (장소)** | 가해자와 대상이 만나는 공간 | 주차장, 골목, 온라인 플랫폼 |

### 외부 삼각형 (3억제자)

| 억제자 | 대응하는 요소 | 역할 |
|---|---|---|
| **Handler (가해자 통제자)** | Offender | 가해자를 개인적으로 아는 사람: 가족, 보호관찰관, 교사 |
| **Guardian (대상 보호자)** | Target | 대상을 지키는 자: 경찰, 경비원, 이웃, CCTV |
| **Manager (장소 관리자)** | Place | 장소를 통제하는 자: 건물주, 관리소장, 상가번영회 |

**핵심 통찰:** 범죄를 막는 가장 효과적인 방법은 억제자 중 **누가 가장 움직일 수 있는가**를 찾아 그를 활성화하는 것입니다.

## SARA의 Analysis 단계에서 사용법

SARA의 A(Analysis) 단계에서 문제를 이 6개 박스로 분해합니다:

```
1. Offender: 누구인가? 동기는? 이곳을 선택한 이유는?
2. Handler: 가해자를 통제할 수 있는 사람이 존재하는가? 작동하지 않는 이유는?
3. Target: 무엇이 공격당하는가? 매력 요인은? (VIVA: Value, Inertia, Visibility, Access)
4. Guardian: 대상을 보호할 자원이 있는가? 왜 작동하지 않는가?
5. Place: 공간의 어떤 특징이 범죄를 가능하게 하는가?
6. Manager: 장소 관리 주체는 누구인가? 협조 가능한가?
```

**개입은 6개 박스 중 어디를 공략할지 선택하는 문제입니다.**

---

## VIVA — 대상 매력도 분석

대상(Target)의 매력을 평가하는 4요소 (Felson):

- **Value** — 금전적·상징적 가치
- **Inertia** — 이동 용이성 (가벼울수록 매력적)
- **Visibility** — 가해자에게 노출된 정도
- **Access** — 접근 용이성

예시: 
- 현금 > 신용카드 (V 동일, I 높음, A 동일)
- 길가 자전거 > 차고 안 자전거 (Vi·A 차이)
- 귀중품 진열 차량 > 빈 차량 (Vi 차이)

---

## CRAVED — 훔치기 쉬운 물건 (Clarke)

특히 절도 대상에 적용되는 확장 모델:

- **Concealable** — 감추기 쉬운가
- **Removable** — 가져가기 쉬운가
- **Available** — 구하기 쉬운가
- **Valuable** — 값나가는가
- **Enjoyable** — 쓰기 좋은가 (특히 사용 장물)
- **Disposable** — 처분하기 쉬운가 (장물 시장)

예: 스마트폰은 CRAVED 전부 해당. 대형 가전은 Removable에서 탈락.

**개입 시사점:** 장물 시장(Disposable) 차단이 재범률 감소에 효과적일 수 있음.

## 개입 옵션 도출법

삼각형 6개 박스 각각에서 개입 가능성을 탐색합니다:

### Offender 측
- 접근금지 명령
- 보호관찰 연계
- 재범 방지 프로그램 연결
- (예방 차원) 교육 프로그램

### Handler 측
- 가족·보호자 연계
- 학교·복지 네트워크 활용
- 시설 관리 주체와 협력

### Target/Victim 측
- 피해자 교육 (V·I 낮추기)
- 표적 경화(Target Hardening) — 잠금장치, 경보
- 보험·보상 체계

### Guardian 측
- 경찰 자원 배치 (최후 수단)
- 민간 경비 유도
- CCTV·감시
- 이웃 감시 네트워크

### Place 측
- CPTED 개입
- 조명, 시야, 접근통제
- 활용도 증가(자연감시 확대)

### Manager 측
- 건물주·관리소와 협약
- 지자체 행정 지도
- 영업자 책임 강화

---

## 한국 경찰 적용 예 — 편의점 강도

**문제:** ○○동 24시간 편의점 연쇄 강도, 심야시간 단독 근무 편의점 집중.

**6박스 분석:**

| 박스 | 분석 |
|---|---|
| Offender | 20~30대, 단독범, CCTV 회피 모자·마스크. 현금 150만원 이하 소액 현장 선호 |
| Handler | 대부분 노숙·주거불안정 상태 → 개인적 통제자 부재 |
| Target | 심야 현금 보유 편의점, 단독 근무자 |
| Guardian | 단독 근무자는 보호자가 아닌 피해자. 112 신고 외 수단 없음 |
| Place | 이면도로 편의점, 진입·탈출 경로 단순, 조명 제한 |
| Manager | 본사 점주협의회, 개별 점주. 방범 책임은 개별 점주에 분산 |

**개입 옵션:**
1. **Place**: 심야 ATM 현금 상한 조정 (Value 감소)
2. **Manager**: 본사 차원 방범 가이드라인, 2인 근무 인센티브
3. **Guardian**: 편의점 간 긴급 공동대응 네트워크
4. **Offender**: 광역 수사로 연쇄성 입증 후 체포

**단순 순찰 강화는 이 구조를 바꾸지 못함.**

---

## 자주 하는 실수

1. **Offender에만 집착** — 체포만이 답이 아님. 나머지 5개 박스가 열려 있으면 새 가해자가 채움
2. **Target/Place 혼동** — 차를 터는 것은 Target, 주차장 설계는 Place. 다른 개입
3. **Guardian ≠ 경찰** — 경찰은 Guardian의 한 종류일 뿐. 지속가능한 Guardian은 주로 민간
4. **Handler 잊음** — 많은 경우 Handler 활성화가 가장 저렴·효과적

## 언제 쓸 것인가

**적합:**
- SARA의 Analysis 단계
- 반복 문제의 원인 분해
- 경찰 실무자가 "왜 안 줄어드는가" 분석할 때
- 개입 옵션이 순찰 강화에만 머물러 있을 때

**부적합:**
- 단일 사건 수사
- 현장 즉각 대응
- 조직범죄·화이트칼라 등 일상활동이론이 약한 영역

## 함께 쓰는 프레임워크

- **SARA** — 본 프레임워크는 SARA의 A 단계 구성요소
- **CPTED** — Place/Manager 개입 설계
- **Hot Spots** — 지리적 집중 확인 후 본 프레임워크로 분해
- **ILP** — Handler·Manager 네트워크 분석
